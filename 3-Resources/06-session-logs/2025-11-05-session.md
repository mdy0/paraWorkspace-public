## Key Decisions

### Use Case Consolidation
- Started with 41 use cases, consolidated to 33 by merging duplicates
- Removed beta reading (added complexity)
- Removed testing & feedback section (managed outside PARA)
- Simplified idea flow: inbox → projects (no intermediate "active ideas" folder)

### Directory Structure
- Numbered folders for workflow sequence (00-status → 08-published)
- Everything-as-folder in inbox (can add sources to ideas later)
- Sources organized: to-read/ (unread) → Category folders (read)
- Categories: AI, Tech, Business, Politics, Culture, Science, Writing
- Session logs in Resources (reference material)
- Metadata folder for system files (inputs, feedback, instructions)

### File Naming Conventions
- Dates: YYYYMMDD format
- Versions: -v1, -v2, etc.
- TK for placeholders (journalist convention)
- Session logs: YYYY-MM-DD-session.md

### Workflow Philosophy
1. Reduce friction - Capture first, organize later
2. Support iteration - Preserve versions, never delete
3. Enable discovery - Find connections across projects
4. Maintain context - Always know where you left off
5. Build consistently - Small daily progress
